# Glossary

| Term | Meaning |
|---|---|
| **Founder Score** | Persistent 5-dimension score of a *person* (Execution, Technical Depth, Problem Insight, Resourcefulness, Momentum). Lives in Memory, never resets, follows the founder across companies |
| **Score band** | Every score is `value [low-high]`, never a point. Width = honest uncertainty. Produced by self-consistency sampling ([[intelligence-layer]]) |
| **Coverage** | 0-1 measure of evidence quality (count, diversity, verified share). Low coverage → wider bands |
| **Visibility Index** | 0-100 *discoverability* (followers, stars, press, accelerator). Deterministic. NEVER feeds capability |
| **Capability−visibility gap** | `composite − visibility`. Large positive = **hidden gem** (the demo protagonist). Large negative = hype case |
| **3-axis score** | Per-*opportunity*: Founder / Market / Idea-vs-Market, each with trend. Never averaged |
| **Trust Score** | Per-CLAIM 0-1 confidence, set by the Validator, with contradictions flagged |
| **Ambition & Drive read** | Idea-agnostic soft assessment: would we back this person if the idea died? ([[research/founder-predictors]]) |
| **Interview Playbook** | Evidence-anchored questions targeting the widest bands, with strong/red-flag answer signatures + expected band reduction |
| **Delta update** | New signal → targeted band adjustment (not full rescore) → appended to ScoreHistory with cause |
| **EvidenceBundle** | Plain-data input to all intelligence stages: founder + signals + claims. DB-independent |
| **ReasoningLog** | Append-only record of every LLM call. Simultaneously the traceability stretch goal and the cache |
| **Cold start** | Founder with deck-only evidence. Response = wide honest bands + the interview that resolves them, never a low score |
| **Hidden gem** | High capability, low visibility — exactly who the system exists to find |
